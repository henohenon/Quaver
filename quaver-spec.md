# Quaver - 仕様書

QRコードから固有のメロディと3Dエフェクトを生成するWebアプリ。
同じ文字列からは常に同じ音が鳴る、決定論的ハッシュベース。

---

## コンセプト

- QR1個 = 1つのメロディ
- スキャンするまで何が鳴るか分からない
- 「世界中のQRには既に音が宿ってる」前提に立つ
- 命名: **Quaver**(英: 8分音符 + 震える)

## 非ゴール(MVP範囲外)

- アカウント / DB / バックエンド
- 楽曲の保存・共有機能(URLハッシュで暫定対応)
- 複数人での合奏 / リアルタイム同期

## 機能要件

### F1. 生成モード
- テキスト入力 → QR表示 + 対応メロディ即再生
- 「もう一回」ボタンで何度でも再生
- 入力が変わると即生成しなおし

### F2. スキャンモード
- カメラ起動 → QRを認識 → 自動再生
- jsQR を `requestAnimationFrame` ごとに実行
- 認識成功時に音 + エフェクト発火

### F3. エフェクト
- 音の進行に同期してQRの黒モジュールが3D空間へ飛び立つ
- 各モジュールが対応する音階の高さに沿って上昇
- 最終的に画面外へ消える(QRが"死ぬ")

---

## 生成アルゴリズム

入力文字列 → SHA-256(32バイト) を「プリセット + 内部変調」の二層に分配。
画一性を避けつつ品質のガードレールを置く構造。

### Tier 1: スタイル選択

**音色スタイル(16種、`bytes[0] % 16`)**

| # | スタイル | 雰囲気 |
|---|----------|--------|
| 0 | Music box | オルゴール、減衰早め |
| 1 | Ambient pad | ロングサステイン、ふわっと |
| 2 | Techno pulse | 4つ打ち + シンセベース |
| 3 | Glitch | bit-crushed、不安定 |
| 4 | Choir | 母音フォルマント、ハーモニー |
| 5 | Bells | 共鳴ストライク |
| 6 | Plucked | 撥弦、ハープ風 |
| 7 | 808 + drums | サブベース + リズム |
| 8 | Granular wash | 粒状音響 |
| 9 | FM piano | 鐘っぽいピアノ |
| 10 | Chip tune | 8bit ファミコン風 |
| 11 | Acid bass | 303 風レゾナンス |
| 12 | Wind chimes | 風鈴のランダム性 |
| 13 | Drone | 持続音 + 倍音シフト |
| 14 | Frog/insect | 自然界の鳴き声系 |
| 15 | Heartbeat | 低音パルス + 余韻 |

**ビジュアルスタイル(8種、`bytes[1] % 8`)**

| # | スタイル | 動き |
|---|----------|------|
| 0 | Crumble | モジュールが音階の高さに上昇 + フェード |
| 1 | Swarm | モジュールが群体で動く(boids的) |
| 2 | Crystal | 結晶化、回転 + 屈折光 |
| 3 | Liquid | 液状化、流動 |
| 4 | Explosion | 全方向爆散 |
| 5 | Constellation | 星座化、線で結ぶ |
| 6 | Origami | 折りたたみ変形 |
| 7 | Decay | 錆びる/朽ちる、粒子化 |

### Tier 2: 内部変調

各プリセット内で以下を変調:

| バイト | 用途 |
|--------|------|
| `[2]` | BPM(60-180) |
| `[3]` | 音数(8-32) |
| `[4]` | スケール選択(各音色スタイルが許容する2-3種から) |
| `[5]` | 密度(疎/中/密) |
| `[6]` | カラーパレット(8種から) |
| `[7]` | カメラワーク(static / orbit / dolly / shake) |
| `[8..31]` | 各音の音階・長さ・ベロシティ |

### 結果

- ベース組み合わせ: 16 × 8 = **128通り**
- 各組み合わせ内で BPM/密度/色/カメラ/音列が変調 → 実質的にどのQRも別物
- 「あ、このQRオルゴール×星座だ」みたいに識別もできる
- スタイルが許容するスケールだけ採用するので、変な音にならない(ガードレール)

### 決定論性

同じ入力 → 同じSHA-256 → 同じプリセット + 同じ変調 → 同じ出力。
ブラウザ標準の `crypto.subtle.digest('SHA-256', ...)` を使用。

### 実装上の構造

```

│   ├── ...
```

## デプロイ

- `vite build` → `dist/` 生成
- GitHub Actionsで自動デプロイ
- `vite.config.ts` に `base: '/quaver/'` を設定(リポジトリ名と一致)
- `gh-pages` ブランチに `dist/` をpush、Pages設定でそれを公開
